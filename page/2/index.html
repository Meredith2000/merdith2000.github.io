在MAC上配置Android Studio

前言

这学期，学校开展了一门android的课，所以不可避免的要安装了Android Studio（下面全部简称AS），但是由于安装的过程中非常的艰难，所以我准备写一篇博客来纪念一下我艰难的安装过程，并帮助一下正在为环境烦恼的同学们。

环境

  • 操作系统：MAC OS
  • IDE：Android Studio

具体步骤

下载AS

这一块很容易，直接在网上搜索就可以了

安装gradle

由于国内的种种原因，无法高速下载gradle，所以我们可以手动安装gradle，首先，先在浏览器中下载gradle，下面是下载地址

https://services.gradle.org/distributions/

然后放在AS创建的.gradle文件下的一串hash值命名的文件夹中，然后再打开AS的设置，选择通过指定路径配置。

配置阿里的maven

由于国内的种种特殊情况，在国内是没办法直接访问google的一些资源库的，但是在构建android app的时候需要下载各种gradle的依赖和库，所以这时候就要借助阿里的maven源，来帮我们构建项目。

这里我想强调一下，这里是一个大坑，因为现在网上百分之八十的阿里的maven源都是过时的，是不能用的，因为阿里已经抛弃的这个接口，我这里给出阿里的maven源地址：

https://maven.aliyun.com/mvn/guide

下面打开build.gradle这个文件，将里面的

google()
jcenter()

替换成：

maven { url ‘https://maven.aliyun.com/repository/google' }
**maven **{ url ‘https://maven.aliyun.com/repository/public' }

第一个源是阿里拉的谷歌的源，相当于google()

第二个相当于jcenter()

更改之后重新加载配置即可享受高速下载的乐趣！

配置host

由于特殊原因，我们无法直接高速的获取google服务器上的sdk，所以我们必须要配置一下host来高速的访问谷歌的服务器，在这里的我想跟大家分享一下我遇到的坑。

网上的很多方法都是使用代理，但是，我发现很多代理都已经失效了，比如说mirrors.neusoft.edu.cn这个网站，我进去看过，实测并没有android的源，但是很多博客都是推荐的这个源，根本就是误人子弟！！所以，还是配置host是最佳方案！！下面废话不多说，直接来教你们怎么配置

  • 首先打开hosts这个文件（路径：/etc/hosts），在打开的时候最好加一下sudo，获取一下最高的权限
  • 在文件中添加

203.208.50.100  dl.google.com
203.208.40.37  dl.l.google.com

  • 保存

现在开始享受高速sdk和模拟器的下载吧！！

基本概念

DBMS的概念

DBMS中文全称：数据库管理系统(Database Management System)

  • DBMS是位于用户与操作系统（OS）之间的一层数据管理软件。
  • 它为用户或应用程序提供访问数据库的方法，包括数据库的建立、查询、更新及各种数据控制。
  • DBMS总是基于某种数据模型，可以分为层次型、网状型、关系型和面对对象型等（）

表和视图区别联系

  • 一个表可以是一个基本表，也可以是一个视图。
  • 基本表是实际存储在数据库中的表，视图是从基本表或者其他输入中导出的表。
  • 视图是一个虚表，它只在数据库中存储其定义，而不存放视图的数据。
  • 视图的数据仍存放在导出该视图的基本表中
  • 视图对应的于关系模型中的外模型，而基本表对应于关系模型的模式。

数据库三级模式结构

概念模式

定义：概念模式（简称模式，也称逻辑模式）是对数据库全局逻辑结构的描述，是数据库所有用户的公共数据视图。

外部模式

定义：外部模式（也称子模式或用户模式）是用户观念下局部数据结构的逻辑描述，是数据库用户（包括应用程序员和最终用户）能够看见和使用的局部数据的逻辑结构和特征的描述

内部模式

定义：内部模式（也称存储模式，内模式）是对数据库中数据物理结构和存储方式的描述，是数据在数据库内部的表示形式。内部模式定义了所有内部记录类型、索引和文件的组织方式，以及所有数据控制方面的细节

关系模型数据约束（完整性规则）

  • 实体完整性规则
  • 参照完整性规则
  • 用户定义的完整性规则

实体完整性规则

关系中元组的主键值不能为空

参照完整性规则

如果属性集K是关系模式R1的主键，同时K又是关系模式R2的外键，那么在R2的关系中，K的取值只允许两种可能：

  • 为空值
  • 等于R1关系中的某个主键值

用户定义的完整性规则

这是针对某一具体数据约束条件，由应用环境决定（比如说筛选学生成绩大于80分的同学等等）

关系代数的五个基本操作

  • 并运算
  • 差运算
  • 笛卡尔积
  • 选择运算
  • 投影运算

并运算

设关系R和S具有相同的元数n（即两关系都有n个属性），相应的属性取自同一个域，则关系R和关系S的并是由属于R或属于S的元组组成，其结果仍然是一个n目关系，记为R∪S，其形式定义为：

R∪S = {t t∈R ∨ t∈S}, t是元组变量

差运算

设关系R和S具有相同的元素n，相应的属性取自同一个域，则关系R和关系S的差是由属于R但不属于S的元组组成，其结果仍是一个n目关系，记为R-S，其形式定义为：

R-S = {t t∈R ∧ t∉S}, t是元组变量

笛卡尔积

设有关系R和S，它们分别是n目和m目关系，即它们的元数据分别为n和m，分别有p和q个元组，则关系R、S经笛卡尔积运算的据结果是一个（m+n）目关系，共有（p×q）个元组，这些元组是由R和S的元组组合而成的。关系R与S的笛卡尔积记为R×S，其形式定义为：

R×S = {t t=⟨tⁿ,tᵐ⟩ ∧ tⁿ∉R ∧ tᵐ∈S}, t是元组变量

选择运算

选择运算是根据某些条件对关系做水平分割，即在一个关系内选择符合条件的元组称为选择运算。选择运算可表示为

𝞼c(R) = {t t∈R ∧ C[t]=True}, 其中𝞼是选择运算符，𝞼c(R)表示从R中挑选满足公式C的元组所构成的关系

投影运算

这个操作是对一个关系进行垂直分割，即对一个关系内属性的指定称为投影运算。选择运算可表示为：

πa(R) = {t t∈R C[t]=True}, 其中π是选择运算符，πa(R)表示从R中挑选满足公式C的元组所构成的关系

保护数据库的安全措施的五个级别

环境级

计算机系统的机房和设备应加以保护，防止有人进行物理破坏。

职员级

工作人员应清正廉洁，正确授予用户访问数据库的权限。

OS级

应防止未经授权的用户从OS处着手访问数据库。

网络级

由于大多数DBS都允许用户通过网络进行远程访问，因此网络软件内部的安全性是很重要的。

DBS级

DBS的职责是检查用户的身份是否合法及使用数据库的权限是否正确。

实体间的几种对应方式

一对一（1：1）联系

若两个实体集中E1、E2中的每一个实体至多和另一个实体集中的一个实体相联系，则称E1和E2是一对一的联系，记为1：1。

例如，学校实体集与校长实体集间的联系是一对一联系。

一对多（1：n）联系

设两个实体集E1、E2，若E1中每一个实体都与E2中的多个实体（包括0个和1个）相联系，而E2中每个实体最多只能和E1中一个实体相联系，则称E1和E2是一对多的联系，记为1：n。

多对多（m：n）联系

设两个实体集E1、E2，若E1中每一个实体都和E2实体集中的多个实体（可以是0个，1个）相联系，反之亦然，则称E1和E2是多对多联系，记为m：n。

超键、主键、候选键的概念

超键

在一个关系中，能惟一标识元组但含有多余属性的属性组合，称为关系的超键。

主键

若一个关系中有多个候选键，则选其中的一个为关系的主键。

用主键实现关系定义中：表中任意两行（元组）不能相同的约束。 主键中任何属性的取值不能为空，主键的值不能重复。

候选键

如果一个属性组能唯一标识元组，且又不含有多余的属性，那么这个属性组称为关系的候选键。

包含在任何一个候选键中的属性称为主属性，不包含在任何键中的属性称为非主属性或非键属性。

执行的结果为：

分析

这道题主要考察的是对队列的理解，我认为要掌握以下的几点

  • 判断队列是否为空
  • 判断队列是否已满
  • 修正指针

判断队列是否为空：

if (head == tail){
// 说明队列为空
}

判断队列是否已满：

if (head == (tail+1)%SIZE){
// 说明队列已满
}

修正指针：

if (head == SIZE){
head = 0;
}

if (tail == SIZE){
tail = 0;
}

代码

#include

using namespace std;

const int NULLVALUE= 999999;
const int SIZE = 10;

class FifoTable{
private:
int table[SIZE]; //存放队列数据的数组
int tail; //队列尾指针
int head; //队列头指针
int max,min; //队列元素的界，也就是如果加入队列的元素值大于max或者小于min，加入操作无法成功。
public:
FifoTable(int max,int min); //结构函数，设置对象的max和min值。
bool AddElement(int d); //向队列加入一个元素d，成功返回真，否则返回假。如果队列满了，则新的值无法加入
int GetElement(); //从队列头获取一个元素，并返回该元素值。如果队列为空，则返回NULLVALUE

};

FifoTable::FifoTable(int max, int min) {
tail = 0;
head = 0;
this->max = max;
this->min = min;
}

bool FifoTable::AddElement(int d) {
// 输入的数据不符合要求
if (d > max d < min) {
return false;
}
// 队列已经满了
if (head == (tail+1)%SIZE) {
return false;
}
table[tail] = d;
tail++;
// 由于是环形队列，所以再tail越界的时候要进行修正
if (tail == SIZE) {
tail = 0;
}
return true;
}

int FifoTable::GetElement() {
int temp;
// 栈空
if (head == tail) {
return NULLVALUE;
}
// 用来暂时保存要返回的值
temp = table[head];
head++;
// 由于是环形队列，所以再head越界的时候要进行修正
if (head == SIZE) {
head = 0;
}
return temp;
}

int main() {
FifoTable ft(20,-2);
int i;

for(i = -2; i <= 23; i++)
    ft.AddElement(i);

for(i = 0; i < 9; i++)
    cout << ft.GetElement() << ",";

cout << ft.GetElement() << endl;
return 0;

}

#include

class polygon
{
private:

this->z = z;

return x+y+z;
}

题目

设计一个Fraction类的结构，该类表示一个可以单步前进的广义整数。该类有以下两个私有属性
int den; //单步前进的步进距离,也就是执行++运算后的增长量；
int num; //当前的整数值
设计和实现该类相应的结构函数，print函数，并重载它的前缀++运算符号和后缀++运算符号。
该类完成后，如下main()函数可以执行。
void main()
{
Fraction b(1,10),a(4,-8);
(a++).print();
a.print();
a = b++;
a.print();
a = ++b;
a.print();
}
执行的结果为
-8
-4
10
12

分析

这道题主要考察的是前置++和后置++运算符的重载

Fraction& operator++(); // 前置++
Fraction operator++(int); // 后置++

后置++运算符必须要在后面加一个int形参，表示与前置++的区别，同时，由于是++运算，所以可以返回自身。前置++运算最好要返回引用，后置可以直接返回指针。

代码

#include

using namespace std;

class Fraction{
private:
int den; //单步前进的步进距离,也就是执行++运算后的增长量；
int num; //当前的整数值

public:
Fraction(int den, int num);
Fraction& operator++(); // 前置++
Fraction operator++(int); // 后置++
void print();
};

Fraction::Fraction(int den, int num) {
this->den = den;
this->num = num;
}

Fraction & Fraction::operator++() {
/*
* 前置++运算符
* 先++，再参与运算
* 在++之后，直接返回自身
*
* */
num += den;
return *this;
}

Fraction Fraction::operator++(int) {
/*
* 后置++运算符
* 先参与运算再++
* 先用一个temp保存要返回的值
* 在自身++
*
* */
Fraction temp(*this);
num += den;
return temp;
}

void Fraction::print() {
cout << num << endl;
}

int main() {
Fraction b(1,10),a(4,-8);

(a++).print();
a.print();

a = b++;
a.print();

a = ++b;
a.print();
return 0;

}

题目

设计一个Complex类的结构，改类表示复数。并设计和实现该类相应的结构函数，show函数，并重载它的+和*运算符号。
该类完成后，如下main()函数可以执行。
void main()
{
Complex c1(3, 5);
Complex c2 = 4;
c1 = c1 + c2;
c2 = c1 * c2;

c2.show();

分析

重载符号结构

重载符号运算符的结构要记好，函数的名字叫做operator，后面加上要重载的符号。最好要传入引用，但是做运算不能叠加在自己的身上，要返回一个新的对象。

class Complex{
private:

int image;

Complex(int real, int image);
Complex(int real);
Complex operator+ (Complex& c); // 重载+号运算符
Complex operator* (Complex& c); // 重载*号运算符
void show();
};

Complex::Complex(int real, int image) {
this->real = real;
this->image = image;
}

Complex::Complex(int real) {
this->real = real;
this->image = 0;
}

// 注意，在下面两个重载中，不能返回自身的引用，要重新返回一个新的对象
Complex Complex::operator+(Complex &c) {
return Complex(this->real + c.real, this->image + c.image);

 \* 这种情况下计算的结果会叠加在c1上，并且返回c1

题目

设计一个名为Arrayint的类的，该类的两个私有属性分别为
int *p; //指向存放整个数组的数据区。
int n; //整个数组的整数个数
p所指向的数据区在结构函数中用动态分配，并在析构函数中释放。
该类可以实现深拷贝。设计和实现该类相应的深拷贝函数和show成员函数，show函显示数组中所有数据。
该类完成后，如下main()函数可以执行。
void main()
{
int a[10] = {5,7,8,9,10,11,12,34,32,22};
Arrayint *x = new Arrayint(10, a);
a[2] = 100;
Arrayint y = *x;
x->show();
delete x;
y.show();
}
执行的结果为:
5 7 8 9 10 11 12 34 32 22
5 7 8 9 10 11 12 34 32 22

分析

这道题主要考察深拷贝的内容。在深拷贝的时候一定要给对象中的指针分配独自的内存空间，比如说下面进行数组的深拷贝。

int *p = new a[10] // 这边就是对p指针进行分配十个int大小的内存空间分配

代码

#include

using namespace std;

class Arrayint{
private:
int *p; //指向存放整个数组的数据区。
int n; //整个数组的整数个数
public:
Arrayint(int n, int *a);
void show();
};

Arrayint::Arrayint(int n, int a[]) {
int i = 0;
this->n = n;
p = new int[n]; // 实现深拷贝，给p分配内存空间
for(i = 0; i < n; i++){
p[i] = a[i];
}
}

void Arrayint::show() {
int i;
for(i = 0; i < n; i++){
cout << p[i] << ‘ ‘;
}
cout << endl;
}

int main() {
int a[10] = {5,7,8,9,10,11,12,34,32,22};

Arrayint *x = new Arrayint(10, a);
a[2] = 100;
Arrayint y = *x;
x->show();
delete x;
y.show();
return 0;
}

题目

设计一个stack类的结构，该类可以完成栈的功能。该栈的元素类型为int，最大深度为100。它能完成push和pop方法，功能分别为压栈和出栈。
如下main()函数可以执行。
void main()
{
stack s1;
int i;
for(i = 1; i < 10;i++)
s1.push(i);
for(i = 1; i <6;i++)
cout << s1.pop()<<”,”;
s1.push(200);
cout << s1.pop()<<endl;
}
执行的结果为
9,8,7,6,5,200

分析

第二题也没有什么难度，主要要注意一下栈的溢出和触底的检验操作

代码

#include

#define maxN 100 // 栈的最大容量

using namespace std;

class stack{
private:
int top; // 栈顶指针，初始化的时候指向-1
int a[maxN]; // 用数组来模拟栈
public:
stack(); // 空参构造器
void push(int num); // push方法
int pop(); // pop方法
};

// 初始化栈
stack::stack() {
top = -1;
}

void stack::push(int num) {
if (top < maxN-1) { // 检验栈有没有溢出
top++;
a[top] = num;
}
}

int stack::pop() {
if (top > -1) { // 检验栈有没有触底
top–;
return a[top+1];
}
return -1;
}

int main()
{
stack s1;
int i;

for(i = 1; i < 10;i++)
    s1.push(i);
for(i = 1; i <6;i++)
    cout << s1.pop()<<",";
s1.push(200);
cout << s1.pop()<<endl;
return 0;

}
