题目

设计一个Complex类的结构，该类表示复数。并设计和实现该类相应的结构函数和add成员函数，该函数完成复数的加法。该类完成后，如下main()函数可以执行。
void main()
{
Complex c1(3, 5);
Complex c2 = 4;
c1.add(c2);
c1.show();
c2.show();
Complex c3(0,5);
c3.show();
}
执行的结果为
7+5i
4
5i

分析

这道题没啥难度，只要是考察构造函数的使用还有类内部方法的设计。

代码

#include

using namespace std;

class Complex{
private:
int real;
int image;
public:
// 构造器
Complex(int real);
Complex(int real, int image);
void add(Complex c);
void show();
};

// 构造器和方法的具体实现
Complex::Complex(int real) {
this->real = real;
this->image = 0;
}

Complex::Complex(int real, int image) {
this->real = real;
this->image = image;
}

void Complex::add(Complex c) {
this->real += c.real;
this->image += c.image;
}

void Complex::show() {
if (real == 0) {
cout << image << ‘i’ << endl;
} else if (image == 0) {
cout << real << endl;
} else{
cout << real << ‘+’ << image << ‘i’ << endl;
}
}

int main()
{
Complex c1(3, 5); // 使用第二个构造器
Complex c2 = 4; // 使用第一个构造器
c1.add(c2); // 传入Complex对象，调用add方法
c1.show(); // 空参，调用add方法
c2.show(); // 同上
Complex c3(0,5); // 调用第二个构造器
c3.show(); // 空参，调用add方法
return 0;
}

jupyter的搭建以及利用nginx对其进行反向代理

Posted on 2020-07-28 Edited on 2020-12-12 In jupyter

前言

这几天一直在忙着学习Vue，一直没有时间更新博客，今天趁着这个机会来一篇关于jupyter搭建的博客，而且网上关于利用nginx对jupyter进行反向代理的文章也比较少，所以我想借此机会分享一下的我经验。

安装环境

  • CentOS 版本： 7.7
  • Nginx 版本：1.16.1
  • Python 版本：3.6.8
  • pip 版本：9.0.3
  • jupyter core 版本：4.6.3
  • ipython 版本：7.16.1

安装步骤

  • 首先安装python3环境（如果安装了就跳过这一步）

yum install python3

  • 使用pip3来安装jupyter（在python3的版本中都是自带pip3的）

pip3 install jupyter

  • 生成配置文件（root用户在后面加上–allow-root）

jupyter notebook –generate-config

  • 使用ipython来生成密码，进入ipython并且输入一下指令，然后进行密码的设置（他返回的秘钥一定要保存好，后面配置文件需要）

from notebook.auth import passwd
passwd()

  • 修改配置文件（root用户的配置文件默认在/root/.jupyter/jupyter_notebook_config.py）

c.NotebookApp.ip = ‘*‘ # 允许所有的ip访问
c.NotebookApp.port = 8888 # 对外暴露的端口号
c.NotebookApp.open_browser = False # 默认不打开浏览器
c.NotebookApp.password = u’sha1:XXXXX’ # 输入刚刚复制的秘钥
c.NotebookApp.notebook_dir = u’/usr/jupyter’ # 设置文件存储的位置

现在在浏览器打开，并输入你的http://ip:port就可以正常访问了

现在基本的安装步骤结束了，但是我这里想要补充一下为jupyter添加ssl证书的步骤：

  • 还是打开jupyter_notebook_config.py这个配置文件，找到并且修改下面的几项

## SSL/TLS 认证文件所在全路径.
c.NotebookApp.certfile = u’你的证书的绝对路径.pem’

SSL/TLS 私钥文件所在全路径.

c.NotebookApp.keyfile = u’你的证书的秘钥的绝对路径.key’

现在，就可以正常开启https服务了。

现在我来说说怎么开启nginx进行反向代理：

  • 我的jupyter是挂在我的一个二级域名下的，但是问题是我的二级域名下面不只有一个网站，所以我要现指定jupyter的目录，打开配置文件：

c.NotebookApp.base_project_url = ‘/你想指定的目录’

  • 指定了jupyter的目录，下面就要使用nginx对jupyter进行反向代理了，打开nginx.conf，修改如下：

location /刚刚指定的目录文件夹/ {
proxy_pass https://你的域名:指定的端口号;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header Host $host;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection “upgrade”;
proxy_redirect off;
}

好了，现在就应该能反向代理了，完美实现隐藏端口号的效果。

Nginx反向代理

Posted on 2020-07-24 Edited on 2020-12-12 In Nginx

前言

昨天一直没有更新博客，是因为我一直在研究Nginx这个Web服务器，我觉得这个Nginx非常有意思，所以我深入的学习了一下。我的初衷是想要通过反向代理来隐藏我的端口号，因为我现在很多不同的网站部署在不同的Web服务器上，但是我只有一个域名，所以我想通过反向代理来隐藏别的服务器的端口号，顺便实现一下所谓的动静分离，所以我使用Nginx在我的服务器端进行了一次极大的改动，所以我想写这篇文章来记录一下。

WordPress

首先我想先说一下WordPress，我原来是搭建了LAMP环境，将WordPress部署在了Apache中，我刚开始的想法是通过Nginx来反向代理，以此来隐藏Apache的端口号，但是问题就在于在WordPress在系统是，存在两种url，分别是：

  • WordPress Address (URL)
  • Site Address (URL)

第一个url是指WordPress后台的地址，是实际的安装地址，这个地址是可以通过Nginx反向代理的，第二个url是用户看到的地址，也就是文章的地址，这一部分地址用Nginx是无法代理的，如果改变了此处的url就会出现文章页面访问不到的情况。

后来，我又想直接在docker中创建一个wordpress，但是还是失败了，因为在访问的时候还是会带有docker中映射的端口号。在这里我想提一下我在使用docker安装wordpress中出现的一个问题，就是我在启动wordpress镜像的时候apache居然报错，原因是apache配置的错误，但是我进入docker，发现里面的设置设置非常简单（果然是简单版的Linux），量最基本的vi都没有，找一个httpd的配置文件都很困难，后来我在网上看到一篇帖子，我按照下面的操作，成功解决了问题：

docker exec -it wordpress /bin/bash # 进入容器的内部
echo “ServerName 你的注意域名” >> /etc/apache2/httpd.conf #把我写的中文用你的域名替换掉
/etc/init.d/apache2 restart

具体配置的路径还要自己探索，我这边只是提供一个思路。

用docker来隐藏端口号失败了，于是我只能整体的迁移的我的wordpress，我说说我整个迁移的思路：

  • 备份WordPress（在设置中可以实现）
  • 安装LNMP环境
  • 启动WordPress

整个过程中唯一的难点就是在Nginx中配置CGI

location / {
index index.php index.html;
}

location ~ \.php$ {
try_files $uri =404;
fastcgi_pass unix:/tmp/php-fpm74.sock; # 这个sock文件需要php-fpm来生成
fastcgi_index index.php;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
include fastcgi_params;
}

好了，至此为止，WordPress迁移已经算是完毕了。

Owncloud

owncloud由于他的后端环境没有wordpress那么复杂，所以他是可以直接通过Nginx来反向代理，来实现隐藏端口号的效果的。具体我是这样配置的：

server {
…….
location / {
return 301 https://$host$request_uri; # http转htpps
}
}

server {
……
location / {
return 301 https://cloud.zkwzkw.top/owncloud; #对cloud.zkwzke.top进行重定向
}
location /owncloud/ {
# 添加访问头，避免owncloud不信任
proxy_set_header Host $host;
proxy_set_header X-Real-IP $remote_addr;
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
proxy_set_header Cookie $http_cookie;
proxy_cookie_path /owncloud/ /;
# 添加最大上传文件的大小
client_max_body_size 5000M;
# 转发路径
proxy_pass 需要转发到的地址:需要隐藏的端口号;
}
}

其实可以从我的配置中看出，我这里是一个妥协的方案，我原本是想当用户访问cloud.zkwzkw.top这个网址的时候，页面直接就可以访问Apache中的wordpress文件，于是我一开始是这样写的：

server{
……..
location / {
https://域名:端口号/owncloud
}
}

但是实际上这样是无法正常访问的，我认为可能是owncloud内部机制的原因。所以我采用了一个折中的方案，就是直接将访问的根目录的请求做一个重定向，重定向到指定位置，在指定的位置，Nginx将会转发请求，并且访问端口号。

总结

到这里，也算勉强的完成了所有页面的端口号的隐藏，不知道各位大佬有啥更好的方案，如果有的话，请在下面留言告诉我，万般感谢！！

Nginx 开启HTPPS

Posted on 2020-07-22 Edited on 2020-12-12 In Nginx

安装环境

  • CentOS 版本： 7.7
  • Nginx 版本：1.16.1

步骤

  • 先将在Nginx的安装目录创建cert文件夹，再将ssl证书和秘钥放入。
  • 打开nginx.conf这个文件，并修改一下内容

server {
listen 443 ssl; #https的访问端口号
server_name localhost; #填写你的域名，例如：www.zkwzkw.top
root html;
index index.html index.htm;
ssl_certificate “填写你证书的路径”;
ssl_certificate_key “填写你秘钥的路径”;
sl_session_timeout 5m;
ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4; #使用此加密套件。
ssl_protocols TLSv1 TLSv1.1 TLSv1.2; #使用该协议进行配置。
ssl_prefer_server_ciphers on;
location / {
root html; #站点目录。
index index.html index.htm;
}
}

斐波那契数列(快速幂求模)

Posted on 2020-07-22 Edited on 2020-12-12 In algorithm

前言

今天在写题目的时候，遇到一个很有意思的题目，题目是关于斐波那契数列的，但是我的脑子里立马出现“矩阵快速幂”，我想肯定是用这种解法，结果折腾了三四个小时，还没搞定。不是因为我写的矩阵快速幂的方法不对，而是这道题无法使用矩阵快速幂来完成。废话不多说了，直接给你们看题目。

题目

带余数斐波那契数公式如下：
fib(n) = (fib(n-1) + fib(n-2)) % r n>=1 r = 173
解析带余数斐波那契数的类结构如下
class fib{
static const int r = 173;
static const int F1 = 1;
static const int F2 = 1;
public:
fib();
//获取fib(n^p)的值 其中n小于1000000，p小于10000，p和n都是正整数
int getfib(int n, int p);
};
以下的main函数执行后
void main( )
{
fib a;
cout << a.getfib(120000,1000) << endl;
}
其输出结果为：

解析

这道题乍一看是使用矩阵快速幂的方法来计算斐波那契数列，但是，这道题坑就坑在，他给出的n非常大，虽然理论上来说通过矩阵快速幂是可以解决很大的n，但是这个n实在是太大，而且用内置的数据类型根本就无法表示，这给后续的运算带来了极大的麻烦。实际上这道题根本就不是真的让你用矩阵快速幂来精确的计算第n项的斐波那契数列的结果，而是一道“找规律”的题目。

在这道题目中，由于对斐波那契数列的结果不断的取模运算，使得斐波那契的结果呈现一个“环状结构”，所以说，根本就不需要计算那么多项的斐波那契数列，知道找到这个“环形结构”的交界点就可以了。

快速幂求模

这是这道题目唯一的难点，就是计算(n^p)%r，解决的思想其实很简单。

首先我们先来了解一下模运算的基本性质：

  1. (a + b) % p = (a % p + b % p) % p
  2. (a - b) % p = (a % p - b % p) % p
  3. (a * b) % p = (a % p * b % p) % p
  4. a ^ b % p = ((a % p)^b) % p

在这里我们重点关注第三条性质。好，我们知道了这个性质之后，让我们再来复习一下小学时候学过的乘法的性质：

代码

#include
#define maxN 100000 // 定义fibCache的大小
#define MOD 173 // 定义取模的大小

using namespace std;

unsigned long long fibCache[maxN] = {0}; // 设置斐波那契数列的缓存结构，将已经计算好的斐波那契的项放在这里面

class fib{
static const int r = 173;
static const int F1 = 1;
static const int F2 = 1;
public:
fib();
//获取fib(n^p)的值 其中n小于1000000，p小于10000，p和n都是正整数
int getfib(int n, int p);
};

// 用来寻找这个循环的周期（大小）
unsigned long long findLoop(){
unsigned long long i = 2;
// 初始化斐波那契数列的缓存数组
fibCache[0] = 0;
fibCache[1] = 1;
// 向febCache中添加值，要保证第一次能够进入，同时在找到“尾巴”的时候退出这个循环
do{
fibCache[i] = (fibCache[i-1] + fibCache[i-2])%MOD;
i++;
}while (!(fibCache[i-2] == 0 && fibCache[i-1] == 1));
// 修正i的值，这时候i表示着斐波那契数列中形成的环的大小
return i-2;
}

fib::fib() {}

// 用来计算最后的落在fibCache的具体位置，并返回结果，这里运用到了快速幂求模
unsigned long long findFib(unsigned long long count, unsigned long long n, unsigned long long p){ // count表示斐波那契数列循环的周期，n表示底数，p表示指数
// res 用来存放最后结果所在位置的索引
unsigned long long res=1,temp=n%count;
while(p){
if (p&1) res=res*temp%count; // p和1进行与运算实际上是判断p的奇偶性，如果p是奇数的话，就会进入这个if，如果是偶数的话，就不会进入这个if（p一定为有一次等于1.就在这时，res获取最后的结果）
temp=temp*temp%count; // 不断地对自己进行平方，并求模
p=p>>1; // 右移一位，相当于除以2
}
return fibCache[res];
}

int fib::getfib(int n, int p) {
unsigned long long count = findLoop();
return findFib(count, n, p);
}

int main() {
fib a;

cout << a.getfib(120000,1000) << endl;

return 0;

}
