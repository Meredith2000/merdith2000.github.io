C++ 期末复习

Posted on 2020-07-21 Edited on 2020-12-12 In C++

前言

最近一直用java和python，都已经渐渐忘记了c++的一些基本用法和概念。但是考虑到马上进学校还有c++的考试，想想还是复习复习吧，于是我就干脆写一篇关于c++重点知识的复习博客，给我自己提提醒，同时可以帮助到正在学习c++的同学们，如果有不足，请多多指正！

知识点总结

拷贝构造函数

拷贝构造函数是一种特殊的构造函数，主要在下面三种情况下会遇到

  • 使用一个已经创建完毕的对象来初始化一个新的对象
  • 将函数对象作为函数的形参
  • 返回值为一个对象

我们先说第一种情况：使用一个已经创建完毕的对象来初始化一个新的对象

person p1(age, name); // 此时p1使用person中的普通的构造函数来完成对象的创建
person p2(p1); // p2的构造是依靠p1来完成的，所以再p2创建的时候就会调用person的拷贝构造函数

下面是第二种情况：将函数对象作为函数的形参

// 在main函数中调用test的时候，p1对象被当做实参传入函数，那么p2就会调用拷贝构造函数来进行初始化。
void test(person p2){
// do something
}

int main(){
person p1(age, name);
test(p1);
return 0;
}

第三种情况：返回值为一个对象

// test函数在返回p1对象的时候，就会调用p2的拷贝构造函数，来初始化p2
person test(){
person p1(age, name);
return p1
}

int main (){
person p2 = test();
return 0;
}

上面已经说完了拷贝构造函数的三种调用的时机，下面就让我来说说拷贝构造函数的写法吧。

class person{
private:
string name;
int age;
public:
// 空参构造器
person(){};
// 参数构造器
person(age, name){
this->age = age;
this->name = name;
}
// 拷贝构造函数
// 引用必须要加，要不然会引起递归调用，编译器会报错
// const 可加可不加，但是建议加上，避免p在函数的过程中发生变化
person(const person& p){
age = p.age;
name = p.name;
}
};

构造函数的调用规则

在默认的情况下，c++编译器会给每一个类添加3个构造函数

  • 默认空参构造函数
  • 默认析构函数
  • 默认拷贝构造函数

但是，一旦用户自定义有参数构造函数，那么编译器将不再自动的提供无参构造函数，但是仍然会提供析构函数和拷贝构造函数。

如果用户自定义拷贝构造函数，那么c++就只会提供默认的析构函数。

Tips:

如果出现以下情况本质上还是调用了拷贝构造函数

person *p = new person(age, name); // 在这里，p并不是一个对象，而是一个指针，没有用到person中的任何构造函数
person p1= *p; // 而在这里，实际上*p所代表的是一个对象，所以p1在初始化的时候调用了拷贝构造函数

深拷贝与浅拷贝

  • 浅拷贝：默认的拷贝构造函数就是浅拷贝，如果遇到指针，只会拷贝指针的地址。
  • 深拷贝：在拷贝指针的时候不是直接拷贝指针的地址，而是在堆区重新开辟一块空间，将原来指针里面的内容拷贝进去，然后在将指针指向新开辟的一块空间（需要程序员自己操作）。

重载运算符

重载运算符定义的格式如下：

person& operator+(person p); // 在class中申明

person& person::operator+(person p){ // 在class外定义
// do something
}

// 前置++运算符的重载
person& operator++(){
}

// 后置++运算法的重载
person operator++(int){
// 为了与前置++运算符的区分，要在后面添加一个int占位符（添加其他的没有用）

// 重载左移运算符必须要在全局进行申明，然后在需要操作的类的内部申明友元函数（具体可以看下面的友元函数的使用）
ostream& operator<< (ostream& cout, person p1){
cout << p1.name;
return cout;
}

Tips:

在使用运算符重载（除了后置的运算符）时，返回值最好是引用数据类型，因为这样可以保证在连续运算的时候操作的是同一个对象

在使用后置的运算符的时候一定不能返回引用，因为后置的运算符是先返回值，再运算的，此时返回的是一个临时的值，只能进行值的传递，要是使用引用的话会有非法访问的问题。

友元函数

在类中具有的private的属性的对象，在类的外面是不可以访问的，而在实际操作中，为了操作的方便，可以在类中申明友元函数，这样，在友元函数中就可以访问该类内部的私有属性。可以参考一下代码来体会一下。

class person{
// 申明是友元函数
friend ostream& operator<< (ostream& cout, person, p1);
private:
string name;
int age;
public:
// 构造函数和方法
};

ostream& operator<< (ostream& cout, person p1){
// 由于在类中申明了是友元函数，所以再改函数中能访问p1中的私有属性，比如说下面的p1.name
cout << p1.name;
return cout;
}

Tips:

友元函数在类类申明的时候要添加friend关键字，但是在全局里面定义的时候前面是不需要加friend的。

  • 公有继承
  • 保护继承

申明和定义的方法：

// 父类
class father(){

}

// 子类，继承父类
class son(): public father{
son(); // 子类构造器
}

// 调用父类构造器的子类构造器
son::son(): father(){
// 里面申明子类构造器
}

tomcat和Apache和谐共处

Posted on 2020-07-19 Edited on 2020-12-12 In Linux

前言

今天遇到一个很莫名其妙的问题，就是在原本正常的tomcat变得无法开启，准确的来说开启的时候什么错都没有报，但是关闭的时候显示端口8005被占用，然后我查看端口的占用情况，发现8005这个端口根本就没有起来。于是我上网查阅资料，发现是因为java熵服务的问题。下面是我的解决方案。

操作环境

  • CentOS 版本： 7.7
  • Apache 版本：2.4.6
  • tomcat 版本： 9.0.36

具体步骤

  • 输入一下指令，安装一下插件，并杀掉所有java的进程。

yum install rng-tools
systemctl start rngd
killall java

  • 这时候重启tomcat就会发现，可以正常起来

其他问题

其实这个时候还是有问题的，因为在这个时候再次打开Apache会发现无法启动。这是因为端口443被tomcat占用了，所以这时候要调整tomcat443的端口。

  • 打开tomcat安装目录/conf/server.xml，将443端口修改成8443，这样就不会有端口冲突的问题了，修改如下

<Connector port="8443" protocol="org.apache.coyote.http11.Http11NioProtocol"
           maxThreads="150" SSLEnabled="true">
    <SSLHostConfig>
        <Certificate certificateKeystoreFile="/usr/apache-tomcat-9.0.36/cert/domain name.pfx" certificateKeystorePassword="Uiqh34u7"
                     type="RSA" />
    </SSLHostConfig>
</Connector>

WordPress 搭建

前言

这个WordPress搭建我总共花了将近10个小时😂，我感觉我把搭建WordPress能遇到的问题都给遇到了。在搭建过程中，我一度想要放弃搭建WordPress，但是最后还是“顺利”的完成了！整个搭建的过程其实说实话不太复杂，但是小的细节非常多。而我用的又是WordPress的最新版本，包括PHP的版（本人是PHP小白，真的一点都不懂），还有Mysql的版本相对来说都是比较新的，所以在搭建的过程中，还有不少的麻烦，下面是我搭建WordPress的主要流程，还有疑难杂症的解决。希望对你有帮助!

安装环境

  • CentOS 版本： 7.7
  • Apache 版本：2.4.6
  • PHP 版本： 7.4.7
  • Mysql 版本：8.0.20

具体步骤

  • 由于我的服务器上没有PHP环境，所以先要安装PHP环境，这里我使用yum安装。

yum install epel-release
rpm -ivh http://mirrors.wlnmp.com/centos/wlnmp-release-centos.noarch.rpm

yum install wphp74

rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

yum install php72w-mysql.x86_64

  • 在一切都安装完成之后，Apache还是无法解析php脚本，这是因为在Apache中还有没对php脚本进行解析，首先先找到Apache的配置文件(httpd.conf)

find / -name “httpd.conf”

  • 找到之后再添加一行配置信息

AddType application/x-httpd-php .php

  • 同时再在此文件之中修改DirectoryIndex，改为

DirectoryIndex index.html index.htm index.php

  • 加载php7_module（如果没有，可以通过yum下载）

LoadModule php7_module modules/libphp7.so

  • 到现在为止，PHP就算是配置成功了。下面开始配置数据库。由于高版本的Mysql在密码的加密上与PHP脚本有说不同，所以我们要改变Mysql中的加密方法。

ALTER USER ‘用户名‘@’%’ IDENTIFIED WITH mysql_native_password BY ‘你的密码’;

  • 下面只要访问你的WordPress网站就行了！

Apache 开启HTTPS

Posted on 2020-07-19 Edited on 2020-12-12 In Linux

安装环境

  • CentOS版本： 7.7
  • Apache版本：2.4.6
  • ssl证书类型：CRT

具体步骤

  • 首先现在Apache的安装目录下建立文件夹，命名为cert。
  • 下载ssl证书。（建议将证书文件、证书链文件、密钥文件分别命名为：domain name_public.crt、domain name_chain.crt、domain name.key）
  • 输入一下命令，来安装mod_ssl

yum install -y mod_ssl

  • 完成安装之后，找到Apache安装目录/conf/httpd.conf，并且打开，找到一下语句

LoadModule ssl_module modules/mod_ssl.so #删除行首的配置语句注释符号“#”加载mod_ssl.so模块启用SSL服务，Apache默认是不启用该模块的。
Include conf/extra/httpd-ssl.conf #删除行首的配置语句注释符号“#”。

  • 找到Apache安装目录/conf.d/ssl.conf，并且打开，修改如下：（注意，在修改的时候由于文件名中间有空格，要用单引号包围）

<VirtualHost *:443>
ServerName #修改为申请证书时绑定的域名www.YourDomainName1.com。
DocumentRoot /data/www/hbappserver/public
SSLEngine on
SSLProtocol all -SSLv2 -SSLv3 # 添加SSL协议支持协议，去掉不安全的协议。
SSLCipherSuite HIGH:!RC4:!MD5:!aNULL:!eNULL:!NULL:!DH:!EDH:!EXP:+MEDIUM # 修改加密套件。
SSLHonorCipherOrder on
SSLCertificateFile cert/domain name1_public.crt # 将domain name1_public.crt替换成您证书文件名。
SSLCertificateKeyFile cert/domain name1.key # 将domain name1.key替换成您证书的密钥文件名。
SSLCertificateChainFile cert/domain name1_chain.crt # 将domain name1_chain.crt替换成您证书的密钥文件名；证书链开头如果有#字符，请删除。

安装环境

  • CentOS版本： 7.7
  • tomcat版本：9.0.36.0
  • ssl证书类型：PFX

具体步骤

  • 首先现在tomcat的安装目录下建立文件夹，命名为cert。
  • 下载ssl证书，并通过ftp传送到刚刚创建的cert目录下。
  • 建议将证书文件名称为domain name.pfx，证书密码文件名称为pfx-password.txt，以便后续操作。
  • Tomcat 9强制要求证书别名设置为tomcat。您需要使用以下keytool命令将protocol="HTTP/1.1"转换成protocol="org.apache.coyote.http11.Http11NioProtocol"。

keytool -changealias -keystore domain name.pfx -alias alias -destalias tomcat

  • 再修改 tomcat安装目录/conf/server.xml中的内容，找到以下的注释，如果没有就复制一个。

    <Connector port=”443” protocol=”org.apache.coyote.http11.Http11NioProtocol”

             maxThreads="150" SSLEnabled="true">
  <SSLHostConfig>
      <Certificate certificateKeystoreFile="证书的路径（建议使用绝对路径）" certificateKeystorePassword="证书的密码（在pfx-password.txt中）"
                   type="RSA" />
  </SSLHostConfig>

  • 重启tomcat，完成。

etc

http自动跳转https

  • 打开conf目录下的web.xml，在后添加以下代码：
CLIENT-CERT Client Cert Users-only Area SSL /\* CONFIDENTIAL

这是我对Linux脚本编写的一点浅薄的认识，如果有错误，请多多指正

关于引号的问题

  • 单引号 纯粹的普通字符
  • 双引号 一般当做普通字符来使用，但是”$” , “`“ , “\“ 仍然被看做是特殊字符
  • 倒引号 被shell解释为可执行的命令

shell中的各种运算符号

  • “$” 是取值运算，是取得变量的值
  • “${ }” 这是将花括号里面的东西看做一个整体

  • “(( ))” 这个里面进行算数运算，相当于let，在进行算数运算的时候引用变量是不需要使用$的

关于位置参数

  • $# 表示输入了几个参数
  • $?可以知道上一次程序执行的情况
  • 用set命令可以在shell中设定参数

set 1 2 3
echo $1 $2 $3

  • 每执行一次shift，命令行上面的参数都会向右移动一位（$0除外）

do
…

do
…
done

# 与上面一种格式完全等价
for i
do
…
done

for ((i=0;i<len;i++))
do
…
done

etc

  • shell中的变量一律当做字符串
  • 在变量赋值的时候要是内容中有空格，一定要用引号引起来
  • [*]是表示将一个数组组合成一个字符串，[@]表示将数组分开输出
  • 在for循环中使用数组最好把in后面的数组用双引号引起来
  • 字符串里面不要放~，shell无法解析
